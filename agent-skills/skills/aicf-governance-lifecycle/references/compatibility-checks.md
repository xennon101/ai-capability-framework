# Compatibility Checks

Compatible changes:

- Description improvements.
- Optional input additions.
- Stricter approval or audit requirements.

Minor-review changes:

- New output fields.
- New enum values when consumers tolerate them.
- More precise model-facing guidance.

Breaking changes:

- Added required inputs.
- Removed or renamed inputs/outputs.
- Weaker risk, approval, audit, idempotency, or tenant requirements.
- Added side effects.
- Lifecycle changes that expose new commit behavior.
