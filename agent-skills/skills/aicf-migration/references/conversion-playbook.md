# Conversion Playbook

1. Split broad tools into narrow capabilities.
2. Add entity manifests where domain objects are shared.
3. Define strict input and output schemas.
4. Add policy, risk, lifecycle, approval, audit, and idempotency metadata.
5. Route only selected read/prepare slices to models.
6. Wire handlers to existing app services.
7. Add deterministic evals and red-team tests.
