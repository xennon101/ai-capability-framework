# Release Checklist

- Branch and worktree state inspected.
- Version, changelog, README, release docs, and package metadata reviewed.
- Generated files are fresh.
- Build, tests, validation, conformance, gates, docs checks, and public checks pass.
- Package dry-run contents are inspected.
- Clean consumer install/import smoke test passes.
- Source archive is produced by repo script when needed.
- Publish action, tag, and dist tag match the intended release.
