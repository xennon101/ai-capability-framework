# Public-Safe Examples

Use synthetic names and simple domains:

- `support.ticket.get`: read a support ticket by ID.
- `support.refund.prepare_case`: prepare a refund request for host approval.
- `support.refund.commit_case`: host-controlled commit for an approved prepared refund.
- `scheduling.invite.prepare`: prepare an invite message.
- `scheduling.invite.send`: host-controlled send path.

Avoid customer-like identifiers, real endpoints, tenant names, transcripts, or provider transport details.
