# Code of Conduct

AI Capability Framework is a public project for building safer AI-accessible application
capabilities. Participation should stay practical, respectful, and focused on improving
the framework.

## Expected Behavior

- Be direct and constructive.
- Assume contributors are trying to improve the public project.
- Keep discussions focused on code, docs, examples, tests, and framework safety.
- Use synthetic examples and public-safe language.
- Respect maintainers when they protect the public repository boundary.

## Unacceptable Behavior

- Publishing credentials, private data, raw prompts, provider payloads, traces, customer
  records, or internal account details.
- Harassment, personal attacks, threats, or discriminatory language.
- Repeatedly pushing changes that bypass safety, validation, or release checks.
- Sharing private drafts or unreleased planning material in public files.

## Reporting

Report security issues through [SECURITY.md](SECURITY.md). For conduct issues, open a
GitHub issue only if it can be discussed publicly without exposing private information.
Otherwise contact the maintainers through the repository owner profile.

Maintainers may remove public content, close issues, or block participation to protect
contributors and the public repository boundary.
