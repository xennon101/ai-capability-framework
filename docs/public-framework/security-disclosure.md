# Security Disclosure

Use [SECURITY.md](../../SECURITY.md) for vulnerability reporting.

Do not publish credentials, raw prompts, provider payloads, traces, customer data,
tenant/account/user IDs, or exploit details in public issues.

Security reports should include:

- affected public version or commit;
- affected capability/runtime/provider surface;
- safe reproduction steps with synthetic data;
- expected impact;
- suggested mitigation if known.

AICF is a framework. Host applications remain responsible for production auth, tenant
isolation, provider credentials, storage, approval identity, and incident response.
